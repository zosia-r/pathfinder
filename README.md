# Train Route Optimization with A*, Dijkstra and Tabu Search

## Project Overview

The goal of the project was to:
- find optimal connections between two stations (shortest time or minimum transfers criterion),
- extend this to multiple stations using Tabu Search.

The graph is built from GTFS data and includes real departure and arrival times, making it time-dependent.

## Algorithms

### Dijkstra
Classic shortest path algorithm adapted to handle:
- time-dependent edges (departure/arrival),
- transfer times between trains.

### A*
Extension of Dijkstra with a heuristic:
- time-based heuristic using haversine distance,
- transfer-based heuristic using train line overlap.

Uses a composite cost `(time, transfers)` or `(transfers, time)` depending on the mode.

### Tabu Search
Used to solve a TSP-like problem over selected stations.

Key features:
- random initial solution,
- neighborhood generated by swapping two stations,
- tabu list to prevent cycling,
- aspiration criterion to allow better solutions even if tabu,
- neighborhood sampling for performance.

Path costs between stations are computed using A* and cached to reduce computation time.


## Usage
- `python main_dijkstra.py ` to find optimal route between two stations using Dijkstra.
- `python main_astar.py ` to find optimal route between two stations using A*.
- `python main_tabu.py ` to find optimal route through multiple stations using Tabu Search.

## Requirements

- Python 3.x
- pandas (for GTFS processing)
